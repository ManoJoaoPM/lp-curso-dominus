# Requisitos da Landing Page Dominus

## 1. Visão Geral
Construir uma Landing Page de alta conversão para a Dominus.
- **Cor principal**: `#ff6600`
- **Logos**: Utilizar as logos na pasta `public/` (`logo-d.png` e `logo-institucional.png`).
- **Framework**: Utilize o comando `npx vite-init . --template react-ts` (ou similar suportado pela ferramenta) para iniciar o projeto React com Tailwind no diretório atual. NÃO crie subdiretórios para o projeto, inicie na raiz atual.

## 2. Design e Estética (Skill Frontend-Design)
- **Estética**: Crie uma interface ousada, moderna e de alta qualidade (production-grade). Fuja de estéticas "genéricas de IA".
- **Tipografia**: Escolha fontes impactantes e distintas (evite Arial, Inter, Roboto). Uma fonte display com muita personalidade para os títulos e uma legível para os textos.
- **Cores**: Fundo escuro/elegante com destaques vibrantes no laranja principal (`#ff6600`).
- **Animações**: Use framer-motion (se React) ou CSS puro para reveals em scroll e microinterações de alta qualidade (delays, fade-ins).
- **Layout**: Quebre a monotonia do grid. Use assimetria, espaços negativos generosos e seções visualmente distintas (ex: fundos texturizados, gradientes sutis).

## Tarefa para o WebDevelopmentTask
1. Inicialize um projeto React+Tailwind usando `npx vite-init . --template react-ts` no diretório atual (sobrescreva/limpe se necessário, mas mantenha a pasta `public`).
2. Implemente a Landing Page inteira dividida em componentes organizados (ex: `src/components/Hero.tsx`, `src/components/FAQ.tsx`).
3. Instale `framer-motion` e `lucide-react` para ícones e animações.
4. Crie um design incrível, moderno e focado em conversão.
5. Inicie o servidor de desenvolvimento e garanta que está tudo funcionando sem erros.
